# Specification: Gemini LLM-Driven Broad Search Gap Audit (`BroadGapLLM`)

## Goal
Improve the Job Application Tracker by implementing an intelligent, large language model (LLM) audited gap analysis tool. The tool runs a broad Gmail search, classifies threads using the `gemini-3.1-flash-lite` model via structured JSON output, compares LLM decisions against our regex filter decisions, and outputs the comparative analysis into a new spreadsheet sheet called `BroadGapLLM`.

This will allow the user to easily identify false skips (missed job emails) and false passes (promotions/newsletters) to iteratively perfect our regex filter.

---

## 1. Authentication Handshake Strategy
To utilize the user's free tier quota securely and conveniently, the script will support two layered authentication methods when contacting the Gemini Generative Language API:

1.  **OAuth Delegation (Primary)**:
    - Add the scope `"https://www.googleapis.com/auth/generative-language"` to `appsscript.json`.
    - Fetch the active user's OAuth access token via `ScriptApp.getOAuthToken()`.
    - Make a REST call to the Generative Language API with an `Authorization: Bearer <token>` header.
2.  **API Key Fallback (Secondary)**:
    - If the OAuth request returns a `403 Forbidden` (e.g. because GCP project linking is not completed), the script will read the `gemini_api_key` property from `PropertiesService.getScriptProperties()`.
    - Append the key to the request URL as `?key=YOUR_API_KEY`.

---

## 2. API Endpoint & Model Details
*   **Model**: `models/gemini-3.1-flash-lite` (or beta endpoint if required, falling back to `models/gemini-1.5-flash` if unavailable)
*   **API Endpoint**:
    - With API Key: `https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash-lite:generateContent?key=<KEY>`
    - With OAuth Bearer Token: `https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash-lite:generateContent`
*   **Method**: `POST`
*   **Content-Type**: `application/json`

---

## 3. Batched structured prompt & JSON schema
To ensure high speed, efficiency, and to prevent Apps Script execution timeouts, we will bundle up to 20 Gmail threads inside a single request. 

### JSON Request Payload Schema
```json
{
  "contents": [{
    "parts": [{
      "text": "System Prompt: You are a strict recruitment and job application classification assistant... [Detailed instructions] ... Input JSON: <Array of threads>"
    }]
  }],
  "generationConfig": {
    "responseMimeType": "application/json",
    "responseSchema": {
      "type": "OBJECT",
      "properties": {
        "results": {
          "type": "ARRAY",
          "items": {
            "type": "OBJECT",
            "properties": {
              "threadId": { "type": "STRING" },
              "isJobRelated": { "type": "BOOLEAN" },
              "classification": { 
                "type": "STRING", 
                "enum": ["APPLIED", "INTERVIEW", "ASSESSMENT", "REJECTED", "OFFER", "NOISE"] 
              },
              "confidence": { "type": "STRING", "enum": ["HIGH", "MEDIUM", "LOW"] },
              "reasoning": { "type": "STRING" },
              "cleanCompany": { "type": "STRING" },
              "cleanTitle": { "type": "STRING" }
            },
            "required": ["threadId", "isJobRelated", "classification", "reasoning"]
          }
        }
      }
    }
  }
}
```

## 4. Spreadsheet Output Sheet (`BroadGapLLM`)
A new tab named `BroadGapLLM` will be created in the bound spreadsheet to render audit results.

### Tab Columns
1.  **Thread ID** (hidden or regular)
2.  **Date**
3.  **From**
4.  **Subject**
5.  **Regex Decision** (`PASS` or `SKIP`)
6.  **Gemini Decision** (`PASS` or `SKIP`)
7.  **Gap Status**:
    - `MATCH`: Both regex and LLM agree.
    - `FALSE_SKIP`: Regex skipped a legitimate application confirmation/update (detected by LLM).
    - `FALSE_PASS`: Regex passed a newsletter/promotion (skipped by LLM).
8.  **Gemini Class**: `Applied`, `Interview Request`, `Assessment`, `Rejected`, `Offer Received`, or `Noise`.
9.  **Gemini Company**: E.g., `Shift Technology`
10. **Gemini Title**: E.g., `Project Manager`
11. **Gemini Reasoning**: The explanation generated by the LLM.
12. **Snippet**: First 150 characters of email body.

## 5. Implementation Modules
1.  **`geminiUtils.gs`**:
    - Coordinates HTTP payloads, headers (OAuth vs. API Key), payload batching, and error fallbacks.
2.  **`main.gs`**:
    - Adds menu item `'Run Gemini Broad Gap Audit'` to the standard UI menu.
    - Defines `runGeminiBroadGapAudit()`, which:
      1. Fetches a 90-day broad Gmail query.
      2. Groups them in batches of 20.
      3. Performs the regex skip check to get the `Regex Decision`.
      4. Runs the batched classification requests to `gemini-3.1-flash-lite`.
      5. Formats the data and writes all rows to `BroadGapLLM` in a single block.
3.  **`appsscript.json`**:
    - Add `"https://www.googleapis.com/auth/generative-language"` to standard OAuth scopes.
4.  **`testFixtures.gs`**:
    - Add custom regression tests to verify payload compilation, parsing of batched responses, and mock OAuth/API key fetch.

---

## Success Criteria
- **Secure Keyless Execution**: Successfully calls the Generative Language API using OAuth credentials if GCP is bound, falling back safely to user's personal key.
- **Batched Performance**: Successfully classifies 100+ threads in under 30 seconds using batched JSON responses.
- **Accurate Gap Identification**: Successfully identifies `FALSE_SKIP` (e.g. Shift Technology) and `FALSE_PASS` (e.g. Indeed marketing) cases side-by-side in `BroadGapLLM`.
